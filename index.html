// Overview

What This Project Does

Standard compiler errors are short and technical. This system converts them into human-readable explanations suitable for students and beginners.

🌳

AST Analysis

Uses Python's built-in ast module to analyze code structure and trace variable usage, answering "What is happening in the program?"

💬

NLG Templates

Rule-based Natural Language Generation converts structured error context into grammatically correct English explanations for beginners.

🖥️

Tkinter GUI

A 700×500 desktop interface lets users paste code, click Analyze, and instantly see beginner-friendly explanations — no terminal needed.

🔍

Regex Extraction

Extracts variable names and line numbers dynamically from live exceptions using regular expressions and the traceback module.

// Source Files

Project Structure

Every file in the repository and its role in the system.

File Role Tag Description
main.py Entry Point CLI Runs the CLI demo with a hardcoded NameError sample. Calls generate_explanation() and prints the result.
explanation_model.py Core Logic Core Contains generate_explanation() and explain_error(). Routes errors to the correct NLG template. Extracts variable names via regex and line numbers via traceback.
gui_app.py GUI Interface GUI Tkinter 700×500 window. User pastes code → clicks Analyze → explanation appears. Uses exec() to run code and catch exceptions.
models/templates.py NLG Templates NLG Defines 4 string templates: NAME_ERROR_TEMPLATE, SYNTAX_ERROR_TEMPLATE, TYPE_ERROR_TEMPLATE, GENERIC_ERROR_TEMPLATE.
models/__init__.py Package Init PKG Makes the models/ directory a Python package so templates.py can be imported via models.templates.
README.md Documentation DOCS Full project overview, viva notes, flow diagram, key concepts, and future improvements.
// Pipeline

System Execution Flow

How a Python exception becomes a human-friendly explanation in 8 steps.

01

User Submits Code

Via CLI sample or the Tkinter GUI input box.

02

Exception Raised

Python runtime raises an Exception during exec().

03

Exception Caught

try/except block captures the live error object.

04

Context Extracted

explain_error() uses regex + traceback to get type, variable, line.

05

AST Dictionary Built

{error_type, variable, line} structured context formed.

06

Template Selected

generate_explanation() matches error type to the right NLG template.

07

Template Formatted

Context values injected into template using .format().

✓

Explanation Shown

Beginner-friendly text printed to console or displayed in GUI.

// Error Types

Supported Errors

Four error categories are handled with tailored explanations.

NameError

Identifies the undefined variable by name, reports the line number, and tells the user to assign a value before use.

SyntaxError

Reports the line with the syntax issue and guides the user to check for missing symbols or incorrect formatting.

TypeError

Identifies the line where a type mismatch occurred and explains that incompatible data types were used together.

Generic

Fallback template for all other exception types. Reports the line and directs the user to review nearby code.

Step 2 — Activate virtual environment
# Windows
venv\Scripts\activate

# macOS / Linux
source venv/bin/activate
Step 3A — Run CLI mode
python main.py
Step 3B — Run GUI mode
python gui_app.py
